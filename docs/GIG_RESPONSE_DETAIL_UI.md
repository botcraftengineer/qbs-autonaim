# Интерфейс детального просмотра откликов на Gig задания

## Обзор

Создан современный, удобный и доступный интерфейс для просмотра и управления откликами на gig задания.

## Компоненты

### 1. ResponseDetailCard (`apps/app/src/components/gig/response-detail-card.tsx`)

Детальная карточка отклика с полной информацией о кандидате.

**Особенности:**
- Полная информация о кандидате (имя, рейтинг, дата отклика)
- Результаты автоматического скрининга с визуализацией оценок
- Табы для организации информации:
  - **Предложение**: цена, сроки, сопроводительное письмо
  - **Портфолио**: ссылки на работы, файлы
  - **Опыт**: опыт работы, навыки
  - **Контакты**: email, телефон, Telegram, профиль
- Быстрые действия: принять, написать, отклонить
- Метаданные отклика

**Доступность:**
- Полная поддержка клавиатуры
- Семантическая разметка
- ARIA-метки для иконок
- Адаптивный дизайн

### 2. ResponseListCard (`apps/app/src/components/gig/response-list-card.tsx`)

Компактная карточка отклика для списка.

**Особенности:**
- Компактное отображение ключевой информации
- Hover-карточка с результатами скрининга
- Быстрый просмотр цены и сроков
- Превью сопроводительного письма
- Быстрые действия прямо из списка
- Переход к детальному просмотру по клику

**Доступность:**
- Кликабельная область ≥44px на мобильных
- Правильная семантика ссылок
- Поддержка Cmd/Ctrl+клик для открытия в новой вкладке

### 3. Страница списка откликов (`page.tsx`)

Обновленная страница со списком откликов.

**Особенности:**
- Поиск по имени кандидата
- Фильтрация по статусу
- Табы: Все, Новые, Оценены, Рекомендованы
- Сетка карточек (адаптивная: 1/2/3 колонки)
- Диалоги подтверждения действий
- Диалог отправки сообщения

### 4. Страница детального просмотра (`[responseId]/page.tsx`)

Новая страница для детального просмотра отклика.

**Особенности:**
- Полная информация об отклике
- Breadcrumb навигация
- Действия: принять, отклонить, написать
- Диалоги подтверждения
- Автоматическая инвалидация кэша

## Маршруты

```
/orgs/[orgSlug]/workspaces/[slug]/gigs/[gigId]/responses
  └─ Список откликов (обновлен)

/orgs/[orgSlug]/workspaces/[slug]/gigs/[gigId]/responses/[responseId]
  └─ Детальный просмотр отклика (новый)
```

## API

Используются существующие tRPC роуты:

- `gig.responses.list` - список откликов
- `gig.responses.get` - получение отклика
- `gig.responses.accept` - принятие отклика
- `gig.responses.reject` - отклонение отклика
- `gig.responses.sendMessage` - отправка сообщения

## Дизайн-система

### Цвета и статусы

**Статусы откликов:**
- NEW - синий (default)
- EVALUATED - серый (secondary)
- INTERVIEW - синий (default)
- NEGOTIATION - серый (outline)
- COMPLETED - зеленый (secondary)
- SKIPPED - красный (destructive)

**HR статусы:**
- INVITE - синий (default)
- RECOMMENDED - зеленый (secondary)
- NOT_RECOMMENDED - серый (outline)
- REJECTED - красный (destructive)
- SELECTED - синий (default)
- CONTRACT_SENT - зеленый (secondary)
- IN_PROGRESS - синий (default)
- DONE - зеленый (secondary)

### Иконки

- User - профиль кандидата
- Star - рейтинг
- Calendar - даты
- Award - скрининг
- Banknote - цена
- Clock - сроки
- FileText - документы
- MessageSquare - сообщения
- CheckCircle2 - принятие
- XCircle - отклонение

## Адаптивность

- **Mobile** (< 768px): 1 колонка
- **Tablet** (768px - 1024px): 2 колонки
- **Desktop** (> 1024px): 3 колонки

## Производительность

- Скелетоны для состояния загрузки
- Оптимистичные обновления UI
- Инвалидация кэша после мутаций
- Ленивая загрузка изображений (если добавятся)

## Доступность (A11y)

✅ Полная поддержка клавиатуры
✅ Семантическая HTML разметка
✅ ARIA-метки для всех интерактивных элементов
✅ Видимые focus-индикаторы
✅ Контрастные цвета
✅ Адаптивные размеры кнопок (≥44px на мобильных)
✅ Поддержка screen readers
✅ Локализованные даты и числа
